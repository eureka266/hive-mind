# PRD Template

默认使用这份中文模板。需求很小或很大时，可以调整章节深度，但默认保持中文标题、中文字段名和中文占位说明。

```markdown
# [功能或项目名称]

## 1. 摘要

- **问题:**
- **目标用户:**
- **为什么现在做:**
- **预期结果:**

## 2. 背景与问题

说明当前现状、为什么不够好，以及今天已经产生的具体摩擦、成本或风险。

## 3. 目标

- 目标 1
- 目标 2

## 4. 非目标

- 非目标 1
- 非目标 2

## 5. 用户与场景

### 主要用户

- 用户是谁
- 权限级别
- 触发场景
- 使用频率

### 关键场景

1. 主路径
2. 关键边界路径
3. 运营、权限或合规路径

## 6. 需求

### 功能需求

- FR1
- FR2

### 非功能需求

- 性能
- 安全与权限
- 日志或可审计性
- 可访问性

## 7. 用户流程

按步骤描述主流程。

## 8. 验收标准

- [ ] 可测试条件 1
- [ ] 可测试条件 2
- [ ] 可测试条件 3

## 9. 数据分析与成功指标

- 先导指标
- 结果指标
- 护栏指标

## 10. 依赖与假设

- 依赖 1
- 假设 1

## 11. 风险与缓解方案

| 风险 | 影响 | 缓解方案 |
| --- | --- | --- |
|  |  |  |

## 12. 发布计划

- 阶段 1
- 阶段 2
- 回滚触发条件

## 13. 待确认问题

只保留会实质阻塞产品、设计、工程、法务或商业化决策的问题。
```

## 验收标准规则

每条验收标准都必须可观察、可测试、无歧义，并映射到具体用户行为或系统行为。

不合格:

- 体验更顺滑。

更好:

- 当用户提交交易查询后，结果页在 300 ms 内展示加载态，并在 5 秒内展示结果卡片或结构化错误状态。
